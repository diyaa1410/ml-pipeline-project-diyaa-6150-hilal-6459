# Concurrency Scorecard

| Question | Mechanism used | Evidence |
|---|---|---|
| **Thread-safe?** | `ConcurrentHashMap` for `results` and `idempotencyIndex`; `AtomicLong` counters in `Metrics`; immutable `Job`/`JobResult` objects passed between threads instead of shared mutable state. | No `synchronized`/manual locks needed anywhere in the hot path — reviewable directly in `Main.java` / `Metrics.java`. Load test (`docs/load-test-results.md`) run at concurrency 80 with no corrupted/lost results. |
| **Visibility guaranteed?** | `JobResult` is immutable and always fully constructed before being published via `results.put(...)`, so any thread reading via `ConcurrentHashMap.get` sees a complete, consistent object (happens-before via the map's internal synchronization) — never a half-updated one. | `model/JobResult.java` — static factory methods only, no setters. |
| **Deadlock-free?** | No nested lock acquisition anywhere in the codebase — the only explicit synchronization primitive is `Semaphore.tryAcquire` (single, non-nested), and Python's `chaos_lock` is a single `threading.Lock` also used non-nested. Everything else is lock-free (atomics, concurrent collections). | Code review: grep for `synchronized`/`Lock` shows a single, non-nested acquisition site in each language. |
| **Liveness guaranteed?** | Worker pool uses `AbortPolicy` (never blocks indefinitely on a full queue); `InferenceClient.classify` uses `tryAcquire(500ms)` instead of blocking forever on the bulkhead; every outbound HTTP call has an explicit `.timeout(2s)`. No thread can wait forever. | `InferenceClient.java` timeout/tryAcquire calls; load test shows the gateway remains responsive (health-check succeeds) throughout an overload run. |
| **Bounded resources?** | `ArrayBlockingQueue(25)` behind the worker pool; `Semaphore(20)` bulkhead on outbound inference calls; latency sample ring buffer capped at 5,000 entries in `Metrics`. No unbounded queue or unbounded thread creation anywhere. | `Main.java` (`QUEUE_CAPACITY`), `InferenceClient.java` (`MAX_CONCURRENT_CALLS`), `Metrics.java` (`MAX_SAMPLES`). |
| **Failure recovery path?** | Timeout + `.handle()` fallback branch on the inference call (degrades to a "neutral" classification instead of failing the request); `AbortPolicy` + explicit 503 on queue overload instead of crashing; graceful shutdown hook drains in-flight work before exit. | `docs/ARCHITECTURE_MEMO.md` Q4 measured numbers; `python-ml-service/server.py` `/chaos` endpoint for repeatable failure injection; demonstrated live in the video. |
